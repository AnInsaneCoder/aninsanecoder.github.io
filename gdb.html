Linux下gdb调试命令总结

Thu 09 April 2015
By Shaohang

编译时在编译选项中加上-g，这样可以将调试信息加入到可执行文件中，然后就可以通过gdb工具调试了。 GDB命令可以通过TAB键补全。 开始GDB调试：

1、gdb name-of-executable-file
2、gdb
    file name-of-executable-file

GDB常用命令： 1、断点

break function-name or filename:function     根据函数名设置断点
break line-number or filename:line-number          根据行号设置断点
clear function          清除函数内所有断点
cearl line-number     清除指定行的断点
delete                         清除所有断点

2、开始和停止

run                         从程序的的最开始开始执行直到遇到断点
continue                    继续执行直到下一个断点
kill                              停止程序的执行
quit                              退出GDB debuger

3、行执行

step                          跳至程序的下一行，如果是函数将会进入函数内部执行
next                         调至程序的下一行，并不会进入函数内部执行
until line-number     继续执行直到给定的行号为止
where                         显示当前的行号以及当前执行到哪个函数

4、源代码

list                              列出源代码
list line-number
list function

5、检查变量

print variable-name     查看变量值
print/x variable-name     以十六进制的方式查看变量，以此类推可以得到各种进制以及数据类型的查看方式

更详细的GDB使用命令，go to http://www.yolinux.com/TUTORIALS/GDB-Commands.html
